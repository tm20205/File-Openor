# なぁにこれぇ？
　ウィンドウの位置、大きさを調節できます。単体での動作が可能です。Python必須ではなくなりました。そして早い！嬉しい！


# adjustor 使い方
　1　File Openor-setting.exeを起動してください。
　2　位置調節機能が有効になっているか確認してください。
　3　無効になっている場合、有効化ボタンを押してください。（押すと「無効化」表示に変化）
　4　設定ボタンを押してください。
　5　ウィンドウの名前や位置等を入れてください。
　　　・特に記述する順番は関係ありません。好きなところに記述してください。
　　　・「adjustorの使い方1.jpeg」画像を参考にしてウィンドウの名前を入れてください。（あくまでも例です。全部が「設定」じゃないよ！！）
　　　・ウィンドウの名前についてのみ日本語（全角）を入力可能です。（もちろん半角英数字等も可）
　　　・その他の位置・幅は、数字（半角）のみでないと受け付けません。エラーで正常に動作しません。
　　　・位置確認ウィンドウを押すと確認用ウィンドウが出てきます。以下簡単な取説。
　　　　　・確認用ウィンドウを設定したい大きさにする。
　　　　　・表示更新ボタンを押す。
　　　　　・それぞれの数値が更新されて、調整したい位置の座標や幅などが一発でわかります。便利！
　　　・移動させたい位置の座標や幅などを入力してください。
　6　適用ボタンを押してください。押さずに閉じると反映されません。（閉じる際に警告は出ないので注意してください。）

※初期値はウィンドウの名前が空欄、位置・幅はすべて「0」です。すべて空欄にして、適用した場合はこれになります。

#増やしたい場合#　
　Ex1　上の作業を繰り返す（開いているところをつかう）

#変更・削除したい場合#
　Ex1　変更・削除したいところ1列を変更・削除するだけ。
　Ex2　適用ボタンを押す。よく忘れるので注意。


# ウィンドウ検索の1ループの待機時間について
　これは、位置調整機能が目的のフォルダー・ソフトを見つけるまで探し続けるのかどうかの設定です。単位はms。（1000で1秒）
　そんなのそんなの重すぎていらないって人は「0」を入れると無効になります。空欄でもいいようにしてますが、あまりよろしくないので無効時は一応「0」を入れたほうがいいとワイトも思います。
　初期値は10ms（ループ有効）


＃ 多重起動防止について
　これは同じフォルダー・ソフトを2つ以上起動しないようにする機能です。事故で一つだけ閉じてしまったときとかに使えるかも。
　初期値は有効です。

# デバッグモードについて
　デバッグモードとは言っていますが.exeは別になっています。(adjustor_debug.exe)
　デバッグモードにするには、Read_Pathフォルダにあるdebug.iniを下の例に習って、直接変更してください。

　～debug.ini～
false
3000
//1行目はデバッグを有効・無効を切り替えます。"true"＝有効、"false"＝無効です。
//2行目はデバッグの表示時間。単位はミリ秒です。
　～debug.ini　おわり～

　初期値はデバッグ無効（false）、3秒表示（3000）です。

　！注意！
　・debug.ini内の1行目や2行目を空欄にしないでください。エラーで起動しなくなります。
　・debug.ini内の1行目は「true」または「false」、2行目は数字（半角）以外入れないでください。正常に動作しなくなります。
